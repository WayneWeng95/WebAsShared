# benchmarks/

Copies of the **application workloads** from the five comparison systems, gathered in one place.
**Micro-benchmarks were not copied** — they are documented in [`MICROBENCHMARKS.md`](./MICROBENCHMARKS.md)
(what each one is and what it measures).

Source overview: [`../benchmark_workloads.md`](../benchmark_workloads.md).

## What's here (application workloads, copied)

```
benchmarks/
├── RMMap/        digital-minist, finra, ml-pipeline, wordcount, wordcount-java
├── RTSFaaS/      MediaReview, SocialNetwork   (+ Env/ configs, dir.sh)

├── Roadrunner/   image-resize, fanout-wasm, fanout-container
```

|--------|------------------|--------|
| **RMMap** | digital-minist (MNIST), finra (compliance), ml-pipeline, wordcount (+ Java) | DAG state transfer |
| **RTSFaaS** | MediaReview, SocialNetwork | transactional stateful apps |
| **Cloudburst** | predserving, mobilenet, summa | ML serving + distributed matmul |
| **Roadrunner** | image-resize, fanout-wasm, fanout-container | Wasm data delivery |
| **Faasm** | *(none local)* | Wasm stateful serverless: ML + Python runtime + cold-start |

## Notes on the copies (dependencies)

These were lifted out of their repos, so some need their original context to actually run:

- **RTSFaaS** — `MediaReview/`, `SocialNetwork/` are `driver.sh`/`worker.sh` launchers. They
  `source ../../dir.sh` and read the `Env/*.env` configs (both copied alongside). The actual
  application logic is Java in the original `RTSFaaS/morph-clients/src/main/java/benchmark/{rdma,socket}`
  (a Maven project — **not** copied; build it from the source repo).
- **Cloudburst** — `predserving.py` / `mobilenet.py` / `summa.py` import `utils.py` (copied) and the
  `cloudburst.*` package; run them via the original Cloudburst install. `server.py` is included for
  the name→benchmark dispatch context.
- **Roadrunner** — examples reference WasmEdge + containerd shims and the `input-data` payloads
  (`file_1M.txt … file_500M.txt`, generated by `make-payloads.sh`) from
  `roadrunner/experiments/evaluation/`; not copied (large/generated).
- **RMMap** — workload dirs include their data files and `service.yaml`/`Dockerfile`; they target a
  Knative + RDMA cluster (see original `RMMap/docs/exp.md`).

## Classification boundary

"Application workload" = runs a realistic app. "Micro-benchmark" = isolates one primitive
(serialization, raw transfer, scheduling/scaling, contention). Cloudburst and Roadrunner were the
judgment calls — see `MICROBENCHMARKS.md` for the reasoning. Adjust if your split differs.
